# Grammar Enforcement for Operational Rules

A Python-based system for strictly enforcing operational design domain (ODD) rules in autonomous systems, as described in the SEAMS 2026 paper.

## Overview

This project implements a grammar-based validation system that ensures operational rules generated by LLMs or other sources adhere to strict safety, syntactic, and semantic constraints.

### Key Features

*   **Type-Safe Grammar**: Uses `Lark` for parsing and `Python` dataclasses for a strongly-typed schema.
*   **ODD Enforcement**: Validates that all variables and constants are within the defined Operational Design Domain (e.g., speed limits, sensor ranges).
*   **Structure Validation**: Enforces complexity limits (depth, predicate count) to ensure verification tractability.
*   **Pre-parse Normalization**: Handles common LLM artifacts like Unicode operators and hidden characters.

    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: .\venv\Scripts\activate
    ```
3.  **Install dependencies**:
    ```bash
    pip install lark
    ```

## Usage

Run the included example script to see the parser and validators in action:

```bash
python -m examples.paper_examples
```

### Defining Rules

Rules are defined using a Python-like syntax supporting:
-   **Variables**: `ego_speed`, `dist_front`, etc.
-   **Arithmetic**: `+`, `-`, `*`, `/`
-   **Relations**: `<`, `<=`, `>`, `>=`, `=`, `!=`
-   **Logic**: `∧` (AND), `∨` (OR)

**Example:**
```text
(dist_front < 5.0) ∧ (ego_speed > 0)
```
